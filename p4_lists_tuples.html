PREREQ P4.1 · LISTS: CREATING & INDEXING

P4.1 Lists: creating & indexing

Numbers and strings hold one value at a time; a list holds many. Lists (and their cousins, tuples) are containers — objects that "contain" other objects — and more precisely sequences: ordered collections of items. Strings and the objects returned by range() are sequences too, so most of what you learn on this page transfers to them for free. The values inside a list are called its elements (or items). The simplest way to create a list is to write the elements between square brackets, separated by commas:

type([]) # list — [] is the empty list [10, 20, 30, 40] # a list of four integers subjects = ['calculus', 'introduction to mathematics', 'computer programming', 'linear algebra']

You reach one element by writing the list's name followed by the element's index — its position number — in square brackets (the subscript operator). Indices start at 0, so a list of four items has 3 as its last index; asking for an index that doesn't exist raises an IndexError, a runtime error:

print(subjects[0]) # calculus — the first element is index 0 print(subjects[3]) # linear algebra — the last index is len − 1 print(subjects[4]) # IndexError — there is no index 4 in a 4-item list

Elements don't have to share a type. A list can hold a string, a float, an integer — even another list. With a nested list, the first index picks an item of the outer list, and a second index reaches inside it:

spam = ['spam', 2.0, 5, [10, 20]] spam[3] # [10, 20] — one index gives you the whole inner list spam[3][1] # 20 — outer index first, then inner

Negative indices count from the end — -1 is the last element, -2 the second-to-last — and len() returns the number of elements, just as it counts the characters of a string:

print(subjects[-1]) # linear algebra print(subjects[-2]) # computer programming len(subjects) # 4
📌 ONE PAIR OF BRACKETS, TWO JOBS

Square brackets are used in quite a few ways in Python, and at first glance creating and indexing look alike. The tell: indexing needs an already-created list to its left (subjects[0]), while creating does not ([10, 20]). So w = [a] builds a new one-element list, but t = a[0] indexes into a.

PREREQ P4.2 · SLICING, CHANGING & COMBINING

P4.2 Slicing, changing & combining lists

An index gets a single value; a slice gets several — as a new list. A slice looks like an index with a colon in it: subjects[n:m] returns the part starting at index n and going up to but not including index m. Leave an index out and Python fills in the obvious default — the beginning, or the end:

subjects[0:3] # first three items — index 3 NOT included subjects[1:-1] # drops the first AND the last item subjects[:3] # same as subjects[0:3] subjects[1:] # same as subjects[1:len(subjects)] subjects[:] # the whole list, as a NEW list — a copy (see P4.6!)

Just like range(), a slice takes an optional third number — the step:

subjects[::2] # every second item (default step is 1) subjects[::-1] # the list in reverse order

Item assignment: lists are mutable

Unlike strings, lists are mutable — you can reassign an element in place. When the bracket operator appears on the left of an assignment, it names the element that will be replaced. This is called item assignment:

numbers = [17, 123, 42, 7] numbers[1] = 5 # item assignment — 123 is replaced print(numbers) # [17, 5, 42, 7]

All in all, you can think of a list as a relationship between indices and elements — a mapping: each index "maps to" one of the elements. (Hold that thought for P5, where dictionaries let you map any key, not just 0, 1, 2, …)

Concatenation, replication & del

The + operator concatenates two lists into a new list, and * with an integer replicates one — exactly as they do for strings. The del statement deletes by index (or by slice), and everything after the deleted value moves up toward the front:

[1, 2, 3] + ['A', 'B', 'C'] # [1, 2, 3, 'A', 'B', 'C'] — a new list ['X', 'Y', 'Z'] * 3 # ['X','Y','Z','X','Y','Z','X','Y','Z'] t = ['a', 'b', 'c', 'd', 'e'] del t[1] # ['a', 'c', 'd', 'e'] — delete by index del t[1:3] # ['a', 'e'] — delete a whole slice
⚠️ SLICE TRAPS
  • [n:m] stops before m — subjects[1:3] holds exactly two items, and alist[4:] runs from index 4 through the last item.
  • A slice always builds a new list — subjects[1:3] is a list even if it holds one element, and subjects[:] is the idiomatic full copy.
  • After del t[1], every later element shifts up one slot — indexes you noted beforehand may now point at different items.
PREREQ P4.3 · LOOPING OVER LISTS

P4.3 Looping over lists

Technically, a for loop repeats its block once for each item in a sequence — iteration by item. That is what for i in range(4) was doing all along: range(4) is a sequence that Python treats like [0, 1, 2, 3] (wrap it in list() to see), so looping over your own list works exactly the same way:

list(range(4)) # [0, 1, 2, 3] — any sequence → list for subject in subjects: # iteration by item — no indexes needed print(subject)

Reading by item is perfect — until you need to write to the elements. Then you need the indexes, and a common Python technique is range(len(someList)). Better still, enumerate() hands you the index and the item at the same time:

numbers = [17, 5, 42, 7] for i in range(len(numbers)): # indexes 0 .. len−1 numbers[i] = numbers[i]**2 # updating needs the index for i, number in enumerate(numbers): # index AND item together print(i, number)

To ask whether a value is in a list at all, skip the loop: the in and not in operators evaluate straight to a Boolean:

print('howdy' in ['hello', 'hi', 'howdy', 'heyas']) # True print('English' not in subjects) # True

And when two lists run in parallel, zip() pairs up the elements at the same index (optional, but handy):

names = ['Bob', 'Sue', 'Amanda'] grade_point_averages = [3.5, 4.0, 3.75] for name, gpa in zip(names, grade_point_averages): print('Name=', name, 'GPA=', gpa)

List comprehensions: a loop in one line

To build the first ten squares you could start from an empty list and append() in a loop — or let a list comprehension combine the loop and the appending into a single line. Add an if clause to keep only the items that pass a test:

squares = [] for value in range(1, 11): squares.append(value**2) # the loop way squares = [value**2 for value in range(1, 11)] # the one-line way list1 = [item for item in range(1, 11) if item % 2 == 0] # evens only
🎯 THREE LOOP PATTERNS — PICK BY WHAT YOU NEED

Just reading? for item in myList. Writing to elements? for i in range(len(myList)). Want both index and item? for i, item in enumerate(myList). Predict the output of each pattern on a 3-item list before you run the notebook cells.

PREREQ P4.4 · LIST METHODS

P4.4 List methods: append, insert, remove, sort, index

A method is a function that is "called on" an object, written after the object with a dot: spam.index('hello'). Each data type has its own set of methods, and the list type ships several for finding, adding, and removing values. append() adds to the end; insert() squeezes a value in at any index; extend() adds each element of another sequence; remove() deletes by value (where del deletes by index):

t = ['a', 'b', 'c'] t.append('d') # ['a', 'b', 'c', 'd'] — in-place! not t = t.append('d') t.insert(1, 'e') # ['a', 'e', 'b', 'c', 'd'] — new value at index 1 color_names = ['orange', 'yellow', 'green'] color_names.extend(['indigo', 'violet']) # adds BOTH elements to the end spam = ['cat', 'bat', 'rat', 'elephant'] spam.remove('bat') # ['cat', 'rat', 'elephant'] — remove by VALUE
⚠️ THE NONE TRAP — AND WHOSE METHOD IS IT?
  • append(), insert(), and sort() return None — they modify the list in place. Write t.append('d'); the "obvious" t = t.append('d') silently replaces your list with None.
  • Methods belong to a single type: 'hello'.append('world') fails, because append() is a list method and strings don't have it.

sort() and index()

sort() orders a list of numbers or strings in place, ascending by default; pass reverse=True for descending. Careful with strings: it sorts in "ASCII order", so every uppercase letter comes before every lowercase one. index() returns the position of a value — the first occurrence if there are duplicates — and raises a ValueError if the value isn't there:

spam = [2, 5, 3.14, 1, -7] spam.sort() # [-7, 1, 2, 3.14, 5] — ascending, in place spam = ['ants', 'cats', 'dogs', 'badgers', 'Elephants'] spam.sort() # ['Elephants', 'ants', ...] — "ASCII order"! spam.sort(reverse=True) # descending order spam = ['Zophie', 'Pooka', 'Fat-tail', 'Pooka'] spam.index('Pooka') # 1 — the FIRST appearance wins spam.index('world') # ValueError — not in the list

A few built-in functions read a whole list at once, no loop required: len(nums), max(nums), min(nums), and sum(nums) — for nums = [3, 41, 12, 9, 74, 15] that's 6, 74, 3, and 154.

PREREQ P4.5 · TUPLES & UNPACKING

P4.5 Tuples: immutable sequences & unpacking

A tuple is a sequence of values much like a list — any types, indexed by integers, usually written in parentheses — with one important difference: tuples are immutable. Indexing and slicing work exactly as for lists, but assigning to an element raises a TypeError:

t = ('a', 'b', 'c', 'd', 'e') print(t[0]) # 'a' — indexing works print(t[1:3]) # ('b', 'c') — slicing works t[0] = 'A' # TypeError — tuples cannot be changed

Strings are immutable in exactly the same way — name = 'Zophie a cat' then name[7] = 't' is a TypeError too. Use a tuple when you want an ordered sequence of values that never changes: it tells anyone reading your code that these values are not meant to be modified. The tuple() and list() functions convert any sequence into the type you need:

t1 = ('a',) # a one-element tuple — the comma makes it t3 = ('a') # just the string 'a'! parentheses alone aren't enough tuple('nsysu') # ('n', 's', 'y', 's', 'u') — any sequence → tuple

Because tuples are immutable, they have no sort() or reverse() methods — but the built-ins sorted() and reversed() accept any sequence and return a new sequence with the same elements reordered, leaving the original untouched.

Unpacking: many assignments in one line

The multiple-assignment trick you met earlier is really tuple unpacking: assign any sequence to a comma-separated list of variables and Python distributes the elements. It is how functions return several values at once — and how the famous one-line swap works:

student_tuple = ('Amanda', [98, 85, 87]) first_name, grades = student_tuple # unpacking def total_ave(grade): total = sum(grade) return total, total / len(grade) # returns BOTH values as a tuple total, ave = total_ave([85, 70, 100, 90]) # 345 86.25 a, b = b, a # the classic swap — unpacking again
⚠️ THE LONELY PARENTHESES

('a') is not a tuple — it's the string 'a' wearing parentheses. A one-element tuple needs the trailing comma, ('a',), or the tuple() function. Predict type(('a')) before you run it.

PREREQ P4.6 · REFERENCES VS COPIES

P4.6 References vs copies: two labels, one list

Technically, variables don't store values — they store references to the memory locations where the values live. The id() function shows the identity (address) behind a name. For immutable values like integers, this detail never hurts you: "changing" spam just re-attaches its label to a brand-new object, and cheese keeps pointing at the old one:

spam = 42 cheese = spam # the REFERENCE is copied — both names point at 42 spam = 100 # spam now refers to a completely different value spam, cheese # (100, 42) — cheese is untouched

But lists are mutable, and that changes everything. Assigning a list to a second variable copies the reference, not the list — so both names refer to the same list object, and a change made through either name is visible through the other:

spam = [0, 1, 2, 3, 4, 5] cheese = spam # the reference is copied, NOT the list cheese[1] = 'Hello!' # this changes THE list — there is only one spam, cheese # ([0, 'Hello!', 2, 3, 4, 5], [0, 'Hello!', 2, 3, 4, 5])
🏷 LABELS, NOT BOXES — THE PAYOFF

P1 asked you to picture a variable as a label attached to an object, not a box containing it. Here is where the picture earns its keep: cheese = spam sticks a second label on the same list. Neither label is "the original" — mutate the object through either one and both see the change, because there is only one object. Immutable values never bite you this way: "changing" them always creates a new object and moves the label.

The same rule explains what methods do. append() doesn't build a new list; it modifies the object in place — the id() stays the same. String concatenation, by contrast, must create a new object, because strings are immutable:

eggs = ['Hello'] eggs.append('World') # in place — id(eggs) is UNCHANGED bacon = 'Hello' bacon = bacon + 'World' # a NEW string object — id(bacon) changes

References also decide what happens when you pass a list to a function: the parameter receives a copy of the reference, so the function can modify your list in place — no return needed (or wanted):

def eggs(someParameter): someParameter.append('Hello') # modifies the caller's list! spam = [1, 2, 3] eggs(spam) print(spam) # [1, 2, 3, 'Hello']

When you genuinely want an independent duplicate, ask for one: the copy module's copy() makes a new list with the same elements (a slice like spam[:] does too), and deepcopy() also copies any lists nested inside:

import copy spam = ['A', 'B', 'C', 'D'] cheese = copy.copy(spam) # a NEW list with a different id cheese[1] = 42 spam, cheese # (['A', 'B', 'C', 'D'], ['A', 42, 'C', 'D']) — independent! # inner lists nested inside? use copy.deepcopy() to copy those as well

One comfort: Python's automatic garbage collector deletes any value no variable refers to any more — memory management is not your job, which removes a whole family of bugs common in other languages.

🎯 THE THREE-LINE RULE
  1. Assignment never copies a list — it copies the reference.
  2. In-place methods (append(), remove(), sort(), …) change the one shared list that every label sees.
  3. Want independence? Take spam[:] or copy.copy(spam) — and copy.deepcopy(spam) when lists are nested inside.
