Explain why Matplotlib is the standard visualization tool of scientific Python, install and set it up, and tell its two plotting interfaces — functional (pyplot) and object-oriented — apart.

Two words carry this whole module. The Figure is the entire window or page — the top-level container that can also hold titles, legends and colorbars. Inside it live one or more Axes: what you actually think of as "a plot", the area where data is drawn, with its own ticks and labels. Every command you will learn manipulates one of these two objects — the only question is who keeps track of them:

Functional (pyplot) interfaceObject-oriented interface
Typical callplt.plot(…), plt.title(…)ax.plot(…), ax.set_title(…)
Who manages figure/axesMatplotlib — stateful: plt remembers the "current" figure and axes and applies every command to themYou — state is explicit in the fig and ax objects you hold
Best forQuick, simple, MATLAB-style plotsIntricate figures, many axes, reuse & fine control
%matplotlib inline # static backend: plots embedded as images in the notebook import matplotlib.pyplot as plt # the universal convention — everyone writes plt plt.style.use('seaborn-v0_8-whitegrid') x = np.linspace(0, 10, 100); S, C = np.sin(x), np.cos(x) # the data to plot # functional style — every plt call targets the CURRENT axes (stateful!) plt.subplot(2, 1, 1); plt.plot(x, S) # (rows, columns, panel number) plt.subplot(2, 1, 2); plt.plot(x, C) # object-oriented style — the state lives in fig and ax, in plain sight fig, ax = plt.subplots(2) ax[0].plot(x, S) ax[1].plot(x, C)

Create scatter plots two different ways — plot() with a marker code or scatter() — choose between them deliberately, and build basic histograms with hist().

A scatter plot shows the relationship between two numeric variables as individual dots. Matplotlib gives you two tools for it, with a real trade-off:

plt.plot(x, y, 'o')plt.scatter(x, y)
Point stylingONE style shared by all pointsIndividual per point — c= (colors), s= (sizes)
Transparency (alpha=)whole line at onceper point (e.g. alpha=0.3)
Colormap support—cmap='viridis' + plt.colorbar()
Speed on large datasetsFaster — the renderer draws one styleSlower — each point built individually

The rule of thumb from the video: for a quick look at big data, plot() wins; when each point should carry extra information through its color or size — a third and fourth dimension! — use scatter() with a colormap:

np.random.seed(42) x, y = np.random.randn(100), np.random.randn(100) colors = np.random.rand(100) # one value per point → mapped to color sizes = 1000 * np.random.rand(100) # one size per point plt.scatter(x, y, c=colors, s=sizes, alpha=0.3, cmap='viridis') plt.colorbar() # show the color scale data = np.random.normal(size=1000) # histograms: the distribution view plt.hist(data, bins=30, density=True) # density=True → total area = 1

Shade the region between curves with fill_between(), and draw polar plots that actually match the definition from your calculus course.

x = np.linspace(0, 2*np.pi, 1000) plt.plot(x, np.sin(x), 'r') plt.plot(x, np.cos(x), 'g') plt.fill_between(x, np.cos(x), np.sin(x), color='red', alpha=0.1) # shade between the two curves t = np.linspace(0, 2*np.pi, 64); r = np.sin(t) plt.axes(projection='polar') # a polar Axes instead of a Cartesian one plt.plot(t + (r < 0)*np.pi, np.abs(r)) # the video's trick: flip negative r by π plt.xticks([0, np.pi/2, np.pi, 3*np.pi/2], ['0', '$\pi/2$', '$\pi$', '$3\pi/2$'])
⚠ POLAR PLOTS VS YOUR CALCULUS BOOK

By default Matplotlib happily plots negative r values on a radial axis that extends below zero — which is not what a math textbook means by r < 0 (there, the point flips to the opposite side of the origin). The fix from the video: plot t + (r < 0)*np.pi against np.abs(r) — add π to the angle wherever r is negative, then take the absolute value.

Reposition and restyle legends, add plain text and arrow annotations, replace the default ticks with your own, and move the spines to build math-textbook-style axes.

Customization is where a figure stops being output and starts being communication. Two tools place words on the plot:

plt.text()plt.annotate()
What it drawsText at a specific (x, y) coordinateText plus an arrow from the text (xytext) to the feature (xy)
Typical useLabeling a curve or a regionPointing at one specific feature (a maximum, an intersection…)
plt.legend(loc='upper left', frameon=True) # where the legend sits + draw its frame plt.text(3.14, -1, 'local minimum', ha='center') plt.annotate('local maximum', xy=(6.28, 1), xytext=(10, 4), arrowprops=dict(facecolor='black')) # arrow: from xytext to xy plt.xticks([0, np.pi, 2*np.pi], ['$0$', '$\pi$', '$2\pi$']) # our own ticks, LaTeX labels ax = plt.gca() # grab the current Axes → switch to OOP ax.spines[['top', 'right']].set_visible(False) ax.spines['bottom'].set_position(('data', 0)) # spines sit at the EDGES by default — ax.spines['left'].set_position(('data', 0)) # move them so the axes cross at the origin

Ticks are the values marking points along an axis; Matplotlib's defaults are usually fine but rarely optimal. Spines are the lines connecting the tick marks that bound the data area — by default they live at the edges of the axes (quiz question 7!), but set_position() can put them anywhere, e.g. through the origin for a textbook look.

Create a whole grid of axes with one plt.subplots(rows, cols) call, index the returned axes array exactly like a NumPy array, and adjust the relative spacing between subplots.

🎬 Interactive: walk the 2 × 3 axes grid in row-major order

One call — fig, axes = plt.subplots(2, 3) — creates all six axes at once and hands them back as a 2-D array. Step through the grid and note the order: rows first, top-left origin, exactly like the NumPy arrays of Module 1:

fig, axes = plt.subplots(2, 3) # (rows, cols) → 2 rows × 3 columns
▸Press "Step ▸" to start
Indexing the axes array
axes[0, 0]top-left subplot
axes[r, c]row r, column c
axes[1, 2]bottom-right
axes.flatiterate row-major
Rows first!

plt.subplots(2, 3) means 2 rows × 3 columns — the same (row, column) convention as a NumPy shape tuple. The first subplot sits at the top-left. This is exactly quiz question 4.

fig, axes = plt.subplots(2, 2) fig.subplots_adjust(hspace=0.4, wspace=0.4) # relative height/width spacing between subplots x = np.linspace(0, 10, 1000) axes[0, 0].plot(x, np.sin(x)) # each cell is a full Axes — any plot type fits axes[0, 1].plot(x, np.cos(x)) axes[1, 0].plot(x, x**2) axes[1, 1].hist(np.random.randn(1000))
