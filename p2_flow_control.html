PREREQ P2.1 · BOOLEAN VALUES & COMPARISON OPERATORS

P2.1 Boolean values & comparison operators

In P1 you learned that a program is a series of instructions. But programming's real strength isn't just running one instruction after another — a program can skip instructions, repeat them, or choose one of several to run. Flow control statements decide which Python instructions execute under which conditions, and they correspond directly to the symbols in a flowchart: diamonds are the branching points, rectangles are the ordinary steps, and rounded rectangles mark start and end.

Before you can write a branching point, you need a way to represent the yes and no options. A Boolean expression is an expression that is either True or False. The == operator compares two operands and produces True if they are equal:

print(5 == 5) # True print(5 == 6) # False

True and False are special Boolean values that belong to the class bool — they are values in their own right, not strings. == is one of the comparison operators (also called relational operators):

ExpressionMeaning
x == yx is equal to y (compares the values)
x != yx is not equal to y
x > y · x < yx is greater / less than y
x >= y · x <= yx is greater / less than or equal to y
x is y · x is not yidentity — do x and y refer to the exact same object in memory?

These operators evaluate to True or False depending on the values you give them — which is exactly what a decision point needs as its condition. Predict each line, then run the cell:

print(42 == 42) # True print(42 == 42.0) # True — it compares the value! print(42 == '42') # False — int/float are always different from a string print(2 != 3) # True print('hello' == 'Hello') # False — Python is case-sensitive print(42 < 100) # True print(42 >= 100) # False
⚠️ = IS NOT ==

Python's symbols differ from the math symbols you grew up with. A single = is the assignment statement; the double == is the comparison operator — mixing them up is one of the most common beginner errors. And there is no such thing as =< or =>: the equal sign always comes second (<=, >=).

One more distinction worth filing away: == asks "are the values equal?", while is asks "are these the same object in memory?". For this refresher, == is almost always the one you want.

PREREQ P2.2 · LOGICAL OPERATORS & PRECEDENCE

P2.2 Logical operators & precedence

The three Boolean operators — and, or, and not — operate on Boolean values and, like comparison operators, evaluate their expressions down to a single True or False. Python evaluates the left expression first, then the right one:

ABA and BA or B
TrueTrueTrueTrue
TrueFalseFalseTrue
FalseTrueFalseTrue
FalseFalseFalseFalse

In words: and needs both sides True; or needs at least one. The third operator, not, simply flips its value: not True → False, not False → True.

print((4 < 5) and (5 < 6)) # True — both sides are True print((6 < 5) or (9 < 6)) # False — neither side is True print((1 == 2) or (2 == 2)) # True — the right side is True print(not (1 == 3) and (3 == 4)) # False — True and False

Like the math operators, the Boolean operators have an order of operations. Arithmetic evaluates first, then the relational (comparison) operators, and the logical operators come last — not before and before or:

LevelCategoryOperators
7 (high)exponent**
6multiplication*, /, //, %
5addition+, -
4relational==, !=, <=, >=, >, <
3logicalnot
2logicaland
1 (low)logicalor
🔍 CHAINED COMPARISONS

Python lets you chain comparisons the way mathematicians write them:

x = 3 0 < x < 5 # True — equivalent to (x > 0) and (x < 5), just more concise

Beware the near-miss versions: x > 0 and < 5 is a syntax error (each comparison needs two operands), and x > 0 or x < 5 is always True — every number satisfies at least one side.

PREREQ P2.3 · BLOCKS & IF / ELIF / ELSE

P2.3 Blocks & if / elif / else

A classic result of computer science says every program can be written with just three forms of control: sequential execution, the selection statement, and the repetition statement — the idea behind structured programming. Flow control statements start with a condition (a Boolean expression, in this context given a more specific name) and are followed by a block of code called the clause or body.

Blocks of code

Python groups lines into blocks, and you can tell where a block begins and ends from the indentation. Three rules: ① a block begins when the indentation increases, ② blocks can contain other blocks, ③ a block ends when the indentation decreases back to zero or to the containing block's level. Find the three blocks in this fragment of a small game:

name = 'Mary' password = 'swordfish' if name == 'Mary': print('Hello, Mary') # block 1 starts here… if password == 'swordfish': print('Access granted.') # block 2 — one line, nested inside block 1 else: print('Wrong password.') # block 3 — also one line
⚠️ INDENTATION IS GRAMMAR

If the statements in one block do not share the same indentation, Python raises an IndentationError before anything runs. The convention: use four spaces per level of indentation, consistently.

Conditional execution: if, else, elif

The if-statement is a compound statement made of the if keyword, a condition, a colon, and an indented block (the if body). The body executes if the condition is True and is skipped if it is False. Adding else gives alternative execution: exactly one of the two branches runs.

if name == 'Alice': print('Hi, Alice.') else: print('Hello, stranger.') # the same choice on one line — the ternary conditional operator (note: no colon!) print('Hi, Alice.') if name == 'Alice' else print('Hello, stranger.')

When there are more than two possibilities, chain conditions with elif ("else, if…"). Python checks each condition in order and runs the body of the first one that is True; the final else catches everything left over:

name = 'Carol' age = 3000 if name == 'Alice': print('Hi, Alice.') elif age < 12: print('You are not Alice, kidd.') else: print('You are neither Alice nor a little kid.') # ← this branch runs
📌 READ IT ALOUD

In plain English an if/elif/else chain reads: "If the first condition is true, do this. Else, if the second condition is true, do that. Otherwise, do the last thing." Trace the Carol example above with your finger: which conditions are checked, and which are never even looked at?

PREREQ P2.4 · WHILE LOOPS, BREAK & CONTINUE

P2.4 while loops, break & continue

To execute a block over and over, use a while statement: the while keyword, a condition, a colon, and an indented body. The body repeats as long as the condition is True:

spam = 0 while spam < 5: print('Hello, world.') spam += 1 # augmented assignment — shorthand for spam = spam + 1
🎯 THE WHILE CYCLE
  1. Evaluate the condition, yielding True or False.
  2. If it is False, exit the loop and continue at the next statement after the body.
  3. If it is True, execute the body — then go back to step 1.

The condition is checked at the start of every iteration, never in the middle. A common pattern keeps a program running as long as the user wants by defining a quit value:

prompt = "\nTell me something, and I will repeat it back to you:" prompt += "\nEnter 'quit' to end the program. " message = "" # empty, so Python has something to check the first time while message != 'quit': message = input(prompt) print(message)

Using break to exit a loop early

The program above works, except it prints the word quit as if it were a real message. The shortcut out is the break statement: when execution reaches it, it immediately exits the loop's body:

message = "" while True: # an infinite loop — the condition is always True message = input(prompt) if message == 'quit': break # the only way out else: print(message)
⚠️ INFINITE LOOPS

A while True: loop with a reachable break is a deliberate idiom — but an infinite loop that never exits is a common bug (typically the body forgets to move the condition toward False). If your program gets stuck, press CTRL-C: it sends a KeyboardInterrupt and stops the program immediately.

continue: back to the top

Rather than leaving the loop entirely, the continue statement skips the rest of the current iteration and jumps back to the beginning of the loop. Here is a loop that counts from 1 to 10 but prints only the odd numbers:

current_number = 0 while current_number < 10: current_number += 1 if current_number % 2 == 0: continue # even → skip the print, back to the top else: print(current_number, end=' ') # → 1 3 5 7 9

By default print() moves to the next line after printing; the end argument changes that — here each value is followed by a single space instead.

"Truthy" and "falsy" values

Python considers 0, None, the empty string "", and empty containers as False in a condition — everything else counts as True:

bool(""), bool(0), bool(None), bool(prompt), bool(12) # (False, False, False, True, True)
🔍 TRUTHY CONDITIONS READ BETTER

That is why while not name: keeps asking until the user types a non-empty name, and if numOfGuests: only prints its reminder when the count is non-zero:

name = '' while not name: # same as: while not name != '' print('Enter your name:') name = input() numOfGuests = int(input('How many guests will you have?')) if numOfGuests: # same as: if numOfGuests != 0 print('Be sure to have enough room for all your guests.')

You could write the explicit comparisons instead — the truthy/falsy forms are simply easier to read once you know the convention.

PREREQ P2.5 · FOR LOOPS & RANGE()

P2.5 for loops & range()

A while loop repeats as long as its condition holds — but what if you want a block to run a certain number of times? That is the for loop statement with the range() function. A for statement has: the for keyword, a variable name, the in keyword, a call to range() (or any sequence — a for loop can iterate item by item), a colon, and an indented body:

print('My name is') for i in range(5): print('Jimmy Five Times (' + str(i) + ')') # Jimmy Five Times (0) … Jimmy Five Times (4) — five iterations
📌 UP TO, BUT NOT INCLUDING

The loop variable i starts at 0 and goes up to — but never includes — the integer passed to range(). That is why range(5) gives five iterations with i = 0, 1, 2, 3, 4. Off-by-one predictions almost always die on this rule.

You can always rewrite a for loop as a while loop — for is just more concise:

i = 0 while i < 5: print('Jimmy Five Times (' + str(i) + ')') i = i + 1

The starting, stopping, and stepping arguments to range()

range() accepts up to three integers: a start, a stop, and a step — the amount the variable increases by after each iteration. A negative step makes the loop count down:

for i in range(12, 16): # start, stop → 12 13 14 15 print(i) for i in range(0, 10, 2): # start, stop, step → 0 2 4 6 8 print(i) for i in range(5, -1, -1): # negative step → 5 4 3 2 1 0 (counts down) print(i)

(Since Python 3.10 the language also has a match statement — a switch-like control statement you may meet in newer code.)

✍️ TRY IT (OPTIONAL — NOT GRADED)

Use nested for loops to print a triangle of asterisks: row 1 has one *, row 2 has two, … up to row 10. Let the outer loop handle each row and the inner loop print each column:

for row in range(__, __): for column in range(__, __): print('*', end='') print()

PREREQ P2.6 · IMPORTING MODULES

P2.6 Importing modules

Every Python program can call the built-in functions — print(), input(), len(), range(), and friends. Python also ships with the standard library: a set of modules, each a Python program containing a related group of functions you can embed in your own programs — math for mathematics, random for random numbers, and so on.

Before using a module's functions you must import it: the import keyword, the module's name, and optionally more module names separated by commas. Once imported, you call its functions with the module. prefix so Python knows where to look:

import random for i in range(5): print(random.randint(1, 10)) # five random integers between 1 and 10

Browse what else ships with Python in the standard library index — a surprising amount of scientific-computing plumbing is already there.

Ending a program early with sys.exit()

Programs normally terminate when execution reaches the bottom of the instructions. To terminate before that, call sys.exit() — it lives in the sys module, so import sys first:

import sys while True: print('Type exit to exit.') response = input() if response == 'exit': sys.exit() # the program ends right here print('You typed ' + response + '.') print('This line will not be printed')
