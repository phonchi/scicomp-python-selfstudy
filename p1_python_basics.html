PREREQ P1.1 · FROM ALGORITHM TO PROGRAM

P1.1 From algorithm to program

Problem solving is the heart of computing, and the solutions you design are algorithms: step-by-step lists of instructions that are well-defined, unambiguous, and guaranteed to finish. An algorithm is like a recipe — it must be followed exactly, and it must end. For a triangle's hypotenuse, the recipe might read:

# Pseudocode — precise English, not yet a program Ask for the first side's length; call it a Ask for the second side's length; call it b Let hypotenuse = √(a² + b²) Display the hypotenuse

Programming is translating that pseudocode into a notation a computer can follow — a program, written in a programming language like Python:

side_a = int(input("Enter the length of the first side:")) side_b = int(input("Enter the length of the second side:")) hypotenuse = (side_a**2 + side_b**2)**(1/2) print("The hypotenuse of that triangle is:", hypotenuse) # → 5.0 for sides 3, 4

Python is an interpreted language: the interpreter reads your statements one line at a time, computing as it goes (a compiler, in contrast, translates the whole program before anything runs). That is why you can type Python into an interactive prompt and get an answer immediately.

Hello, World — more going on than you'd think

print('Hello, World!')
  • print() is a built-in function — a pre-defined operation. The value between the parentheses is its argument.
  • 'Hello, World!' is a string constant — fixed data, written with single or double quotes. The quotes mark where the string begins and ends; they are not printed.
  • print('Hello', 'World') prints multiple values separated by spaces; a trailing \ lets a statement span two lines.

Strings come with methods — actions attached to the data itself, called with a dot: 'hi python'.title() → Hi Python, plus .upper(), .lower(), and .strip() (trim surrounding whitespace). And everything to the right of a # is a comment — notes for humans that Python ignores.

🔍 PREDICT, THEN RUN

The single best habit for learning to read code: predict what a snippet will output before running it, then run it and compare. Where your prediction fails is exactly where your mental model needs repair — the same habit this course asks of you with AI-generated code.

PREREQ P1.2 · NUMBERS, STRINGS & OPERATORS

P1.2 Numbers, strings & operators

An expression combines values and operators to compute a new value — 2 + 3 is an expression; so is (a**2 + b**2)**(1/2). Python's arithmetic follows the precedence rules you know from math, with two operators that surprise newcomers:

OperatorMeaningExampleResult
+ − * add, subtract, multiply7 * 856
/true division — always float45 / 411.25
//floor division — drop the fraction45 // 411
%modulus — the remainder45 % 41
**exponentiation (NOT ^ — that's bitwise XOR)2 ** 101024
⚠️ TWO CLASSIC TRAPS
  • 18.0 // 4 → 4.0, not 4 — floor division truncates, but mixing a float in makes the result a float.
  • 16 - 2 * 5 // 3 + 1 → 14 — precedence: 2*5 first, then 10//3 → 3, then left to right. When in doubt, add parentheses.

The same operators do double duty on strings: + concatenates and * replicates:

s = 'hello' + 'world' # 'helloworld' t = s * 4 # 'helloworldhelloworldhelloworldhelloworld'
PREREQ P1.3 · VARIABLES & TYPES

P1.3 Variables & types

A variable stores a value for later use. You create one with an assignment statement — name, equals sign, value — and Python creates it the first time you assign to it. Assigning again overwrites (the old value is forgotten):

spam = 'Hello' # 'Hello' is a string object… spam = 'Goodbye' # …now spam refers to a different object
🏷 LABELS, NOT BOXES

In Python everything is an object, and a variable is best pictured as a label (reference) attached to an object — not a box containing it. Unlike algebra, where x = 3 for the whole derivation, a Python name can be re-attached to a new value at any time — even a value of a different type. This picture pays off again in P4, when two labels end up on the same list.

Naming rules: names use letters, digits, and underscores, cannot start with a digit (current_balance ✔, 4account ✘, current-balance ✘), and are case-sensitive — size, Size, and SIZE are three different variables. A few reserved keywords (if, for, class, …) can never be variable names — help('keywords') lists them.

Every value has a type

type('Hello!'), type(False), type(4), type(3.2), type(3+5j) # (str, bool, int, float, complex)

You convert between types two ways. Explicitly, with a cast: int(3.9) → 3 (truncation — no rounding!), float(3) → 3.0, str(3) → '3', int('3') → 3. Implicitly, when you mix numeric types in one expression — Python widens to the more precise type (5 + 4.2 → 9.2), following the ladder bool → int → float → complex. Two small friends worth remembering: ord('A') → 65 and chr(65) → 'A' convert between a character and its Unicode code point.

PREREQ P1.4 · DEBUGGING

P1.4 Debugging: three kinds of bugs

You will spend a real fraction of your programming life finding and fixing errors — debugging. It goes much faster once you can name what kind of bug you are looking at:

KindWhat went wrongExampleHow it announces itself
Syntax errorthe code breaks Python's grammarprint(5 + )refuses to run at all — message points at a line
Runtime errorlegal code hits an impossible situation while running5 / 0 · pront(5)crashes mid-run: ZeroDivisionError, NameError, …
Semantic errorthe program runs — and computes the wrong thing'3' + '5' → '35'silence — no message at all

The classic semantic bug in beginner code: input() always returns a string, even when the user types a number. Forget the conversion and + quietly concatenates:

num1 = input('Enter a number:') # user types 3 → '3' (a string!) num2 = input('Enter another number:') # user types 5 → '5' print(num1 + num2) # '35' — concatenation, not addition print(int(num1) + int(num2)) # 8 — convert first ✔
🎯 THE DEBUGGING LOOP

Semantic errors give no error message, so you must supply the alarm: decide the expected output for a known input (a test case) before running, then compare. When they disagree, print() the intermediate values until you find where reality and expectation part ways. Syntax and runtime errors are kinder — read the message: it names the line and the problem.

PREREQ P1.5 · YOUR FIRST SCRIPT

P1.5 Your first script & the Zen of Python

The interactive prompt is for experiments; real programs live in a script — a plain text file of statements, run top to bottom. This little program uses everything from this page:

"""This program says hello and asks for your name and age.""" # multiline comment print('Hello, world!') myName = input('What is your name? ') # input() returns a string print('It is good to meet you, ' + myName) # + concatenates strings print('The length of your name is:\n' + str(len(myName))) # \n = newline myAge = input('What is your age? ') print('You will be ' + str(int(myAge) + 1) + ' in a year.')

Trace the conversions in the last line from the inside out: input() gives a string → int() makes it a number so + 1 is arithmetic → str() makes it a string again so + can concatenate. len() returns the number of characters in a string. In Jupyter, the magics %%writefile hello.py and %run hello.py save and execute a script without leaving the notebook.

The Zen of Python

Type import this in any Python prompt and you get the language's design philosophy as a poem. Two lines are worth adopting as a student: "Simple is better than complex" and "Readability counts." Don't try to write perfect code — write code that works, check it, and only then decide whether it is worth polishing. (When you are ready for the community's formatting conventions, the reference is PEP 8.)
