PREREQ P6.1 · STRINGS ARE SEQUENCES

P6.1 Strings are sequences

Text is one of the most common forms of data your programs will handle. You already know how to concatenate two strings with + — but you can do much more: extract partial strings, add or remove spacing, convert letters to lowercase or uppercase, and check that strings are formatted correctly. First, the basics: the simplest way to create a string is to enclose the characters in single, double, or triple quotes:

type(''), type(""), type("""""") # (str, str, str) — all three quote styles work print("I'am fine") # a benefit of double quotes: a single quote can live inside

A string is a sequence that maps an index to case-sensitive characters — so everything you can do to a sequence, you can do to a string. Access one character at a time with the bracket operator, count characters with len(), and count backward from the end with negative indices:

fruit = 'banana' fruit[1] # 'a' — "b" is the 0th letter, "a" the 1st, "n" the 2nd len(fruit) # 6 fruit[-1], fruit[-2] # ('a', 'n') — negative indices count from the end

Slicing works too, with the familiar [start:stop:step] pattern: you get the characters from start up to but not including stop. Omit the first index and the slice starts at the beginning; omit the second and it runs to the end. Predict each line before you run it:

s = 'Cool-Python' print(s[:5]) # same as s[0:5] → 'Cool-' print(s[5:]) # same as s[5:len(s)] → 'Python' print(s[::2]) # same as s[0:len(s):2] → every 2nd character print(s[::]) # same as s[:] → copy the string print(s[::-1]) # same as s[-1:-(len(s)+1):-1] → reverse the string
📌 ONE MENTAL MODEL, MANY TYPES

Index, negative index, slice, len(), in, and the for loop all work on a string exactly as they work on a list — a string is a sequence, just one whose elements are characters. (Internally, strings are stored as contiguous blocks of memory representing the character sequence — a very memory-efficient layout.)

Immutable — you build new strings, you never edit old ones

s = "hello" s[0] = 'y' # TypeError — strings are immutable and cannot be modified

The best you can do is create a new string that is a variation on the original — and id() proves it really is a different object:

print(id(s)) # the object's address… s = 'y' + s[1:len(s)] # build a NEW string from pieces print(id(s)) # …a different address: a different object print(s) # 'yello'
⚠️ IMMUTABLE MEANS IMMUTABLE

s[0] = 'y' raises a TypeError — you cannot assign into a string the way you assign into a list. This same fact returns in P6.4: methods like upper() return a new string; they never change the original in place.

Traversal and the in operator

A lot of computations process a string one character at a time: start at the beginning, select each character in turn, do something to it, continue until the end. The traversal loop is exactly the one you know:

# Test if s contains 'o' b = False for char in s: # retrieve characters one by one if char == 'o': b = True break print(b)

For a simple membership test, though, in and not in do the whole loop for you and evaluate to a Boolean:

print('o' in 'Hello') # True print('cats' not in 'cats and dogs') # False
PREREQ P6.2 · ESCAPES & RAW STRINGS

P6.2 Escape characters & raw strings

An escape character lets you use characters that are otherwise impossible to put into a string: a backslash (\) followed by the character you want to add. (Despite consisting of two characters, it is commonly referred to as a singular escape character.) The escape for a single quote is \' — so a single quote can sit inside a single-quoted string without ending it:

spam = 'Say hi to Bob\'s mother.' # the \ tells Python: this quote is not the end

The full starter kit: \' single quote, \" double quote, \\ a literal backslash, \t tab, and \n newline (line break). One print statement can exercise them all — predict the output first:

print("Hello there!\nHow are you?\n\tI\'m doing fine.") # Hello there! # How are you? # I'm doing fine.

Triple-quoted strings are the third quoting style from P6.1 — a string written with ''' or """ can span several lines, which is handy for longer text (you will see a multiline letter get split('\n') into lines in P6.4).

Raw strings: when the backslash is the point

Place an r before the opening quote to make a raw string. A raw string completely ignores all escape characters and prints any backslash that appears in it:

print(r'That is Carol\'s cat.') # That is Carol\'s cat. — the backslash survives path = r'C:\Users\Al\Desktop' # raw strings shine for Windows file paths

Because the string is raw, Python treats the backslash as part of the string, not as the start of an escape character — a lifesaver whenever a string naturally contains many backslashes.

PREREQ P6.3 · STRING FORMATTING

P6.3 Putting strings inside other strings

Putting strings inside other strings is a common operation in programming. So far you have done it with + and string concatenation — which works, but requires a lot of tedious typing (and a str() call for every number):

name = 'AI' age = 33 language = 'Python' print("Hey! I'm " + name + ", " + str(age) + " old and I love " + language + " Programing")

A simpler approach is string interpolation. The first tool is the format operator %: the first operand is the format string, which contains one or more format specifiers saying how the second operand should be formatted — %s for a string, %d for an integer ("d" for decimal). One benefit: no str() calls needed.

print("\nHey! I'm %s, %d years old and I love %s Programing" % (name, age, language)) # like printf in C
🔍 ONE SYMBOL, TWO JOBS

Applied to integers, % is the modulus operator (45 % 4 → 1). But when the first operand is a string, % is the format operator, and the result is again a string. Same symbol, completely different meaning — the type on the left decides.

Format specifiers also control layout: the numbers after % give the total field width and (after a .) the width of the decimal part. Values shorter than the field are padded with spaces — by default Python right-aligns numbers and left-aligns other values such as strings. %f formats floating points, and note that b gets rounded:

a = 32 b = 32.145 print('a=%4d, b=%6.2f' % (a, b)) # a= 32, b= 32.15 print(f'a={a:4d}, b={b:6.2f}') # a= 32, b= 32.15 — same result, f-string style

f-strings: the modern default

Python 3.6 introduced f-strings (the f is for format): like string interpolation, but braces replace %, and the variables — or whole expressions — go directly inside the braces. Like raw strings, f-strings take a one-letter prefix before the opening quote:

print(f"\nHey! I'm {name}, {age+2} years old and I love {language} Programing") # inline arithmetic works!
ApproachExampleNotes
+ concatenation"I'm " + name + ", " + str(age)every number needs str(); tedious to type and to read
% format operator"%s is %d" % (name, age)printf-style; no str() calls needed
f-string (3.6+)f"{name} is {age+2}"expressions live right inside the braces — the modern default

F-strings give you the same width control and more. The format specifier now sits after the variable, separated by a colon; <, >, and ^ pick left, right, and center alignment; a + before the width forces a sign on positive numbers (negative numbers always start with -); a 0 before the width (after the +, if any) fills the field with zeros instead of spaces:

print(f'[{a:<15d}]') # [32 ] — < left-aligns in a width-15 field print(f'[{b:^9.2f}]') # [ 32.15 ] — ^ centers print(f'[{a:+10d}]') # [ +32] — + forces the sign print(f'[{a:+010d}]') # [+000000032] — 0 pads with zeros

See the format-spec mini-language for every option; yet another route is the format() method, covered in this Real Python guide.

PREREQ P6.4 · STRING METHODS

P6.4 The string-method toolbox

Strings are Python objects: they carry both data (the characters) and methods — functions built into the object, available to every string. dir(s) lists them all, help() gives quick documentation, and the best reference is the official string-methods page. Here is the working set, grouped by job: case validate search split / join.

upper() and lower() — and why reassignment matters

spam = 'Hello, world!' spam = spam.upper() # 'HELLO, WORLD!' — a NEW string, assigned back spam = spam.lower() # 'hello, world!'

These methods do not change the string itself — they return new string values. To change the original, call the method and assign the result back to the variable: spam = spam.upper(), never just spam.upper(). (Just as eggs + 3 does not change eggs, but eggs = eggs + 3 does — immutable data cannot be modified in place.) Their killer application is the case-insensitive comparison: it should not matter whether the user types Great, GREAT, or grEAT:

print('How are you?') feeling = input() if feeling.lower() == 'great': # normalize case before comparing print('I feel great too.') else: print('I hope the rest of your day is good.')

The isX() family — questions about a string's nature

Several methods whose names begin with is return a Boolean describing the string: isupper()/islower() (at least one letter, all upper/lowercase), isalpha() (letters only, not blank), isalnum() (letters and numbers only, not blank), isdecimal() (numeric characters only, not blank), isspace() (only spaces, tabs, and newlines, not blank), and istitle() (each word an uppercase letter followed by lowercase letters). Predict all seven outputs before running:

print('Hello, world!'.islower()) # False — the H is uppercase print('hello, world!'.islower()) # True print('hello'.isalpha()) # True — letters only print('hello123'.isalnum()) # True — letters and numbers print('hello123'.isdecimal()) # False — not numeric characters only print(' '.isspace()) # True print('This Is Title Case'.istitle()) # True
🎯 THE VALIDATION LOOP

The isX() methods earn their keep validating user input — keep asking until the answer has the right shape:

while True: print('Enter your age:') age = input() if age.isdecimal(): # a valid number? escape the loop break print('Please enter a number for your age.')

Two more Boolean questions ask about the ends of a string: startswith() and endswith() return True if the string begins or ends with the string you pass in. And replace() is the "search and replace" of a word processor:

print('Hello, world!'.startswith('Hello')) # True print('abc123'.endswith('12')) # False — it ends with '123' greet = 'Hello Bob' nstr = greet.replace('Bob', 'Jane') # 'Hello Jane'

join() and split() — taking text apart, putting it back

join() turns a list of strings into a single string: it is called on the separator, gets passed the list, and returns the concatenation with the separator inserted between each item. split() is its inverse — called on a string, it returns a list of strings:

print(', '.join(['cats', 'rats', 'bats'])) # 'cats, rats, bats' print(' '.join(['My', 'name', 'is', 'Simon'])) # 'My name is Simon' ' and '.join(['cats', 'rats', 'bats']) # 'cats and rats and bats' 'My name is Simon'.split() # ['My', 'name', 'is', 'Simon'] — default: any whitespace 'cats, rats, bats'.split(',') # ['cats', ' rats', ' bats'] — a custom delimiter
🔍 JOIN LIVES ON THE GLUE

The string join() is called on is what gets inserted between the list items: ', '.join(['cats', 'rats', 'bats']) → 'cats, rats, bats'. It is separator.join(list) — not list.join(separator). By default split() splits wherever whitespace (space, tab, or newline) appears, and those whitespace characters are not included in the returned pieces.

A common use of split() is chopping a multiline (triple-quoted) string along its newline characters — one list item per line:

spam = '''Dear Alice, How have you been? I am fine. There is a container in the fridge that is labeled "Milk Experiment." Please do not drink it. Sincerely, Bob''' spam.split('\n') # a list in which each item is one line of the letter

strip(), lstrip(), rstrip() — tidying the edges

Sometimes you want to strip whitespace characters (space, tab, and newline) off a string's edges. strip() returns a new string with the whitespace removed from both ends; lstrip() and rstrip() handle the left and right ends only:

spam = ' Hello, World \n' spam.strip() # 'Hello, World' spam.lstrip() # 'Hello, World \n' spam.rstrip() # ' Hello, World'

You will use indexing, slicing, and string methods in almost every Python program you write.
