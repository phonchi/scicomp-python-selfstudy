Explain how symbolic computation differs from numerical computation, install and configure SymPy, and use it as an exact, arbitrary-precision calculator with constants like sp.pi, sp.E and sp.oo.

Symbolic mathematics (also called computer algebra) manipulates equations as symbols — the way you do on paper. Ask ordinary Python for 1/3 and you get 0.3333333333333333, a rounded 64-bit float. Ask SymPy and you get the exact rational 1/3, which stays exact through every later step:

Numerical (float / NumPy)Symbolic (SymPy)
1/30.3333333333333333 — roundedRational(1, 3) — exactly ⅓
√21.4142135623730951 (≈16 digits, fixed)sqrt(2) — exact; evalf(100) → 100 digits
π, e, ∞finite float approximations onlyexact objects: sp.pi, sp.E, sp.oo
Result of 0.1 + 0.2 − 0.35.55e-17 — a round-off residueexactly 0
Strengthspeed on large arraysexactness & algebraic manipulation

The working principle of the whole module: SymPy keeps everything exact until you ask for digits — and then evalf() can produce as many digits as your memory allows (arbitrary-precision arithmetic).

import sympy as sp # the conventional alias sp.Rational(1, 3) * 6 # 2 — exact rational arithmetic, no rounding sp.Float(3.53) + sp.Float(3) # 6.53 — Float has ARBITRARY precision (memory-limited) (sp.pi + sp.exp(1)).evalf() # 5.85987448204884 — exact until YOU ask for digits sp.sqrt(2).evalf(100) # 1.414213562373095048801688724209… (100 sig. digits) sp.oo > 999999 # True — even infinity is a first-class object

Create symbols with sp.Symbol() / sp.symbols(), build expressions from them, and transform those expressions with expand(), factor(), subs() and simplify() — plus control the order in which terms are printed.

A symbol is a placeholder that never evaluates to a number until you substitute one. Combine symbols with operators and you get an expression — SymPy stores it as-is and never simplifies or expands unless explicitly asked:

x = sp.Symbol('x') # the name is given as a string x, y = sp.symbols('x y') # several at once (sympy.abc is a third way) p = (x + y) ** 2 # stays (x + y)**2 — nothing happens automatically p2 = sp.expand(p) # x**2 + 2*x*y + y**2 sp.factor(p2) # (x + y)**2 — back again p2.subs({x: 1, y: 2}) # 9 — a dict maps each symbol to its value sp.simplify(p2.subs({x: 1 - y})) # 1 — substitute an EXPRESSION, then simplify

The four manipulation verbs

FunctionWhat it doesExample
expand()multiply everything out into a sum of terms (trig=True unlocks trig identities)(x + y)² → x² + 2xy + y²
factor()the opposite — rewrite as a product of factorsx² − 1 → (x − 1)(x + 1)
simplify()general "make it simpler" heuristic — tries many strategiessin²x + cos²x → 1
subs()replace a symbol by a value or another expression; returns a NEW expression(x² + 3x + 1).subs(x, 2) → 11

Expressions are immutable

Every manipulation returns a new expression — the original is never modified. That is why you write p2 = sp.expand(p) and keep both.

🖨 PRINTING ORDER — LEX VS REV-LEX

By default SymPy prints a polynomial's terms in lexicographical order (order='lex'): descending powers of x, highest first. Call sp.init_printing(order='rev-lex') to reverse it — ascending powers, constant first: 1 + y + 2x² + x²y. This only changes the display, never the mathematics. (The quiz loves this one.)

Solve single equations and systems of equations either symbolically with solve() or numerically with nsolve() / nroots() — and pick the right solver for each job.

SymPy assumes every expression you pass to a solver equals zero, so x² + 2x − 8 = 0 is solved as sp.solve(x**2 + 2*x - 8). The symbolic solver returns exact roots in a list; when no closed form (an exact-formula answer) exists, the numerical fallbacks take over:

x = sp.Symbol('x') sp.solve(x**2 - 2) # [-sqrt(2), sqrt(2)] — EXACT roots, in a list y = sp.Symbol('y') sp.solve((x + y - 6, x - y - 2), dict=True) # a system → [{x: 4, y: 2}] (dict=True labels each root) sp.nsolve(sp.cos(x) - x, x, 1) # 0.739085… — NUMERICAL, needs a starting point (here 1) sp.nroots(x**3 + 2*x**2 - 1) # ALL numerical roots — polynomials only

The naming rule

lowercase limit / diff / integrate → evaluate now.
Capitalized Limit / Derivative / Integral → build an unevaluated object; evaluate later with .doit().
evalf() → turn an exact result into a float.

sp.limit(sp.sin(x)/x, x, 0) # 1 — the classic indeterminate form, resolved exactly sp.limit((1 + 1/x)**x, x, sp.oo) # E — limits at infinity work too sp.Limit(1/x, x, 0, dir='-').doit() # -oo — approach 0 from the LEFT t = sp.Symbol('t'); St = 5*t**2 # position of a falling object over time sp.diff(St, t) # 10*t — first derivative of St w.r.t. t sp.diff(St, t, 2) # higher order: the 3rd argument is the order
⚠ DEFAULT DIRECTION IS '+'

Unless the limit point is ±∞, sp.limit() approaches the point from the positive side (dir='+') by default. For a left-hand limit you must pass dir='-' explicitly — sp.Limit(1/x, x, 0, dir='-') gives −∞ while the default gives +∞. A favorite trap in the test bank.

Compute indefinite, definite and improper integrals, expand a function into a Taylor series (and strip the O(…) term with removeO()), and convert any expression into a fast numerical function with lambdify().

sp.integrate(x**2, x) # x**3/3 — indefinite: note, NO "+ C"! sp.Integral(sp.sin(x), (x, 0, sp.pi/2)).doit() # 1 — definite: pass (x, a, b) sp.integrate(sp.exp(-x), (x, 0, sp.oo)) # 1 — improper: infinity is a legal bound s = sp.series(sp.cos(x), x, 0, 10) # 1 - x²/2 + x⁴/24 - … + O(x**10) s.removeO() # drop O(x**10) so the polynomial can be evaluated f = sp.lambdify(x, s.removeO(), 'numpy') # symbolic → fast NumPy function f(np.linspace(0, 1, 100)) # evaluate 100 points at once — no loop needed
⚠ WHERE IS "+ C"?

integrate() returns one antiderivative, not the whole family — the constant of integration is omitted because any constant works. If your problem needs it (e.g. an initial-value problem), add your own C = sp.Symbol('C').

Three ways from symbols to numbers

ToolInput → outputUse it when…
subs()symbol ↦ value → new (exact) expressionone-off substitution, staying symbolic
evalf(n)exact expression → Float with n digitsyou want digits for one result
lambdify()expression → real Python/NumPy functionevaluating MANY points fast (plots, tables)

lambdify() is the bridge back to Module 1: do the mathematics symbolically, then hand the final formula to NumPy for fast array evaluation — no loop or list comprehension required.

Construct sp.Matrix objects (from a list of rows, or with eye / zeros / diag) and perform exact matrix algebra — products, inverse, determinant, eigenvalues — even with symbols as elements.

a, b, c, x, y, z = sp.symbols('a b c x y z') m1 = sp.Matrix([[a, b, c], [x, y, z], [a, b, c]]) # list of ROWS — just like NumPy m2 = sp.diag(a, b, c) # diagonal matrix; sp.eye / sp.zeros also exist m1 * m2.T # TRUE matrix product (element-wise needs multiply_elementwise()) 3*m1 + m2 # scalar multiple and sum — element-wise m1 + 2 # ✘ Error! SymPy matrices do NOT broadcast m2.inv(), m2.det(), m2.eigenvals() # exact inverse, determinant, eigenvalues
